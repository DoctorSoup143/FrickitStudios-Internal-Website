Change In Tasking

All task updates and reviews will now be communicated through GitLab. As we slowly transition out of using Discord as our primary method of contact, we are beginning to implement a more streamlined tasking and review system- while also incorporating threaded topic based replies. By moving all task updates and communication to GitLab, we now have all information related to a task, or issue, in a thread based reply system making it easier for managers to track progress and quality check during reviews. This implementation removed ClickUp from the developers hands further reducing the platforms required to obtain all necessary information to complete a task. The new tasking process is as follows:

- Project Managers create a task (issue) in GitLab, add a description, and assign developers.
- Developers create a branch from the new issue upon starting the task.
- Any and all communication regarding the task is sent through the tasks threaded reply system.
- When the task is complete, the Developer creates a merge request, lists all changes from commits on the source branch, assigns themselves, and assigns Sean as the reviewer.
- The Project Manager and Sean review the task and close once approved.

GitLab Wiki Implementation

As time progresses, all concepts will be moved to the Wiki section of the SIGMA GitLab page. This change will allow developers to get more information on any system without awaiting a response or a document. The wiki page can be found at the bottom left of the sidebar once inside the project folder.
